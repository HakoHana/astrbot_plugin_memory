# Memory 插件更新日志

## v0.3.0 (2026-06-09)

### 修复
- 修复索引检查中图边外键完整性验证未自动清理悬挂边的问题
- 补充 `target_node_id` 的悬挂边检查（仅检查了 `source_node_id`）
- 图边完整性检查现支持自动删除指向不存在节点的悬挂边

### 优化
- 所有存储模块初始化异常会被记录但不再阻塞插件启动
- 索引检查改为后台异步执行，不阻塞主流程

## v0.2.0 (2026-06-06)

### 新增
- 图谱引擎实体共现统计 (`entity_cooccur`)
- `relates_to` 关系自动升级机制
- 用户分层系统（core/active/occasional/new）
- 消息预过滤机制（噪声消息丢弃）
- 写操作日志与崩溃恢复 (`WriteOpLog`)
- WebUI 页面 API 注册
- 归档模块冷数据归档 (`Archiver`)
- 指令系统 (`/日记`, `/记忆`, `/记忆搜索` 等)

### 修复
- 修复 `on_llm_request` 中指令消息跳过 LLM 的日志与处理逻辑
- Agent Tools 注册失败不再阻塞初始化

## v0.1.0 (2026-06-01)

### 新增
- 初始版本：日记式长期记忆插件
- 日记捕获与自动摘要
- 记忆原子存储与 FTS5 全文检索
- 人设引擎与周期性人设更新
- 基础图谱节点与边存储
- 记忆注入（system_prompt + user message）
- 数据库迁移框架 (`DBMigration`)
