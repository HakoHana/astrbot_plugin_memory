# 记忆系统架构设计

> 基于 AstrBot 平台的日记式长期记忆插件设计，融合 Obsidian 的笔记理念与 LivingMemory 的工程实践。

---

## 一、核心理念

### 双轨记忆模型

```
用户对话 ──→ 日记（情感叙事，人可读）──→ 原子事实（结构化，机器可搜）
                                      └──→ 全局事实表（去重 + 强化）
                                      └──→ 知识图谱（关联发现）
                   └──→ 用户画像（人格沉淀）
```

三条产出线同步生成，互相关联。

### 设计原则

| 原则 | 说明 |
|------|------|
| 自描述 | 日记内容 = YAML frontmatter + Markdown 正文，导出即 Obsidian 笔记 |
| 去重优先 | 相同内容的事实全局唯一，反复提及自动强化重要度 |
| 图谱副产物 | 写日记时解析 `[[实体]]` 自动建图，零独立调用 |
| 用户可辨 | 每条消息带 sender_id + sender_name，LLM 能分清谁说了什么 |

## 三、数据模型

### 3.1 日记（diary_entries）

```sql
CREATE TABLE diary_entries (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     TEXT NOT NULL,
    date        TEXT NOT NULL,
    content     TEXT NOT NULL,          -- frontmatter + Markdown
    topics      TEXT DEFAULT '[]',
    sentiment   TEXT DEFAULT '',
    importance  REAL DEFAULT 0.5,       -- MAX(原子重要度)
    atom_count  INTEGER DEFAULT 0,
    created_at  REAL NOT NULL,
    updated_at  REAL NOT NULL,
    status      TEXT DEFAULT 'active'
);
```

**content 格式**：

```markdown
---
date: 2026-06-07
mood: 开心
importance: 0.85
topics: ["技术讨论", "bug修复"]
entities: [Hako, GPT-SoVITS]
---

今天修复了 napcat 的 bug... [[Hako]] 担心我消失
```

### 3.2 全局事实（atomic_facts）

```sql
CREATE TABLE atomic_facts (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    content      TEXT NOT NULL UNIQUE,   -- 全局唯一，自动去重
    atom_type    TEXT NOT NULL,
    importance   REAL NOT NULL DEFAULT 0.5,
    confidence   REAL NOT NULL DEFAULT 0.7,
    source_count INTEGER NOT NULL DEFAULT 1,  -- 被引用次数
    created_at   REAL NOT NULL,
    updated_at   REAL NOT NULL
);
```

**去重强化**：提取新事实时查 `content`，已存在则 `importance += 0.05`，不存在则 INSERT。

### 3.3 日记-事实关联（diary_fact_links）

```sql
CREATE TABLE diary_fact_links (
    diary_id   INTEGER NOT NULL,
    fact_id    INTEGER NOT NULL,
    importance REAL DEFAULT 0.5,
    snippet    TEXT DEFAULT '',
    PRIMARY KEY (diary_id, fact_id)
);
```

多对多：一条日记引用多条事实，一条事实被多篇日记引用。

### 3.4 用户注册表（user_registry）

```sql
CREATE TABLE user_registry (
    user_id        TEXT PRIMARY KEY,   -- QQ号，不变
    user_name      TEXT NOT NULL,      -- 最新显示名，可改
    first_seen_at  REAL NOT NULL,
    last_seen_at   REAL NOT NULL,
    name_updated_at REAL NOT NULL
);
```

### 3.5 消息表（messages）

```sql
CREATE TABLE messages (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id  TEXT NOT NULL,
    role        TEXT NOT NULL,         -- user / assistant
    content     TEXT NOT NULL,
    sender_id   TEXT DEFAULT '',       -- QQ号
    sender_name TEXT DEFAULT '',       -- 当时的名字（历史快照）
    timestamp   REAL NOT NULL
);
```

## 四、核心流程

### 4.1 消息处理流水线

```
用户发消息
  │
  ├─→ 提取 sender_id + sender_name → 注册 user_registry
  │                                  → 存储 messages
  ├─→ 检测 /指令 → 处理并阻止 LLM 响应
  ├─→ 记忆注入 → 召回原子 → 拼入上下文
  └─→ consolidation_manager（计数 → 判断→ 触发整理）
```

### 4.2 L1 整理流水线

```
触发整理
  ├─ 1. Judge：LLM 判断值不值得记
  ├─ 2. Diary：LLM 写日记（frontmatter + [[实体]]）
  ├─ 3. Atoms：LLM 提取关键事实（第一人称）
  ├─ 4. Dedup：查 atomic_facts → 强化或 INSERT
  │           → diary_fact_links 关联日记
  ├─ 5. Graph：解析 [[实体]] → 建节点和边
  └─ 6. 重要度 = MAX(原子重要度)
```

### 4.3 用户识别

```
event.get_sender_name()
  ├─ event.sender_name
  ├─ event.message_obj.sender.nickname
  └─ user_id（兜底）
       ↓
  messages.sender_name = 当时名字（历史快照） ← LLM 上下文用这个
  user_registry.user_name = 最新名字（可覆盖）  ← 查询用这个
```

### 4.4 重要度衰减（每日）

```
atomic_facts.importance *= 0.99（可配置）
memory_atoms.importance *= 0.99
```

不提的自然降权，反复提的因去重强化而加分。

### 4.5 搜索排序

```
FTS5 召回 → 候选池 → 按 (重要度×0.6 + 匹配度×0.4) 重排
```

权重可配置。

## 五、数据流向

```
用户 ──→ on_message
  ├─→ messages ── 历史
  ├─→ user_registry ── 名字
  ├─→ capturer
  │     ├─→ diary_entries
  │     ├─→ atomic_facts（去重）
  │     ├─→ diary_fact_links
  │     └─→ graph_nodes/edges
  └─→ retriever → injector → LLM
```

## 六、对比 LivingMemory

| 维度 | LivingMemory | 本系统 |
|------|-------------|--------|
| 存储 | FAISS + BM25 + SQLite | 纯 SQLite |
| 日记 | 无 | Daily Note + frontmatter |
| 原子 | parent_memory_id | atomic_facts（全局唯一） |
| 去重 | Jaccard + reinforce | UNIQUE + Jaccard |
| 图谱 | graph_entries 中间表 | [[实体]] 自动解析 |
| 过期衰减 | 无 | 每日衰减（可配置） |
